Purpose
This specification describes the intended behaviour before implementation begins.

1. Business Overview

Partial Novation enables transfer of part of an OTC trade while retaining the remaining position.

2. Scope

  • Create Partial Novation request.
  • Capture novation amount.
  • Validate mandatory fields.
  • Submit request for booking.

3. Functional Behaviour

  • User selects an eligible OTC trade.
  • User enters partial amount.
  • System validates mandatory inputs.
  • System books the novated trade.

4. Validation Rules

  • Trade Date cannot be later than Business Date.
  • Novation Amount must be greater than zero.
  • Mandatory fields must be populated.

5. UI Behaviour

  • Settlement Date is editable.
  • User can modify all entered values before submission.
  • Validation messages appear below the affected field.

6. Error Messages

  • Trade Date cannot be greater than Business Date.
  • Novation Amount is mandatory.

7. Out of Scope

  • Bulk Partial Novation.
  • Cross-currency support.
  • Audit reporting enhancements.

8. Known Limitations

  • Supports single trade only.
  • No retry mechanism for failed booking requests.
